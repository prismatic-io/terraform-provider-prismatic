# Pulumi acceptance harness

The Pulumi analog of the Terraform acceptance suite. `pulumi_acc_test.go` (`TestPulumiAcc`) consumes
this provider through the [Pulumi Terraform bridge](https://www.pulumi.com/registry/packages/terraform-provider/)
and runs a real **create → preview (no-diff) → destroy** round-trip against a **live Prismatic API**,
using [`pulumitest`](https://github.com/pulumi/providertest).

It bridges the **freshly-built local provider binary** (built to `../bin` by the test), so it exercises
the exact code in your working tree — not a registry-published version. `pulumitest` manages a local
(`file://`) state backend, the config passphrase, `pulumi install` (which regenerates the bridged SDK
from `Pulumi.yaml`), stack creation, and teardown, so there's no Pulumi Cloud account and nothing to
set by hand.

## Run it

The test self-skips unless `PULUMI_ACC` is set (which `mise run testpulumi` does) and requires live
credentials. It creates and destroys REAL resources.

```sh
# Sourcing credentials from the prism CLI:
PRISM_TARGET=testpulumi mise run testacc:prism

# Or with credentials already exported:
export PRISMATIC_URL=... PRISMATIC_REFRESH_TOKEN=...
mise run testpulumi
```

The Pulumi CLI, Go, and Node all come from mise (`mise install`).

## What's committed vs generated

Committed: `pulumi_acc_test.go`, `index.ts`, `Pulumi.yaml`, `package.json`, `tsconfig.json`, this README.

Generated by the test (git-ignored): `sdks/prismatic/` (regenerated from `Pulumi.yaml` by `pulumi
install`), `node_modules/`, and `Pulumi.<stack>.yaml`.

The program exercises `prismatic_organization_signing_key`, whose generated Pulumi token is
`prismatic:index/organizationSigningKey:OrganizationSigningKey`. If the provider schema changes,
re-check the class/alias in the generated `sdks/prismatic` and update `index.ts`.

The test asserts a full create → read → destroy round-trip **and** that a second plan is empty — the
same post-apply idempotency Terraform/OpenTofu guarantee. That requires deterministic inputs, so
`index.ts` uses a fixed public key; a freshly generated key per evaluation would make every plan
re-plan a `RequiresReplace` change (in Pulumi and Terraform alike).
